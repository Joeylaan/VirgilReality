## Project Overview

**Client:** VirgilReality
**Website:** https://virgilreality.nl/
**Project Type:** Portfolio & Service Showcase Website Redesign

---

## About VirgilReality

**Who They Are:**
VirgilReality is a Dutch video production company specializing in professional filmmaking and post-production services.

**Their Tagline:**
_"Where your vision becomes reality"_

**Services Offered:**
- Music videos
- Aftermovies (event recaps/highlights)
- Online course content
- Social media videos
- Original content creation
- Full-service production (directing, cinematography, editing)
- Consulting on larger productions

**Their Promise:**
Complete film productions from A to Z, entirely customized to client needs. They position themselves as a full-service production partner rather than offering piecemeal solutions.

---

## Website Purpose

The website serves as:
1. **Portfolio showcase** — displaying completed projects and past work
2. **Service offering** — communicating what they do and how they work
3. **Lead generation** — converting potential clients into inquiries
4. **Brand presence** — establishing authority and professionalism in video production

---

## Type of Website

**Category:** B2B/B2C Portfolio & Service Marketing Site

**Key Components:**

## Agent Roles for This Redesign

### 1. **Plan Agent** — Architecture & Implementation Planning
**Role:**
Designs the overall implementation strategy for the website redesign. This agent will:
- Analyze the current site structure and identify gaps
- Create a step-by-step implementation plan
- Identify dependencies between different components
- Recommend technology stack and architectural decisions
- Plan feature priorities and phasing

### 2. **UI/UX Designer Agent** — Visual Design & User Experience
**Role:**
Creates the visual direction and user experience for the redesigned site. This agent will:
- Design wireframes and mockups for all website sections
- Create a cohesive, modern design system that reflects VirgilReality's creative brand
- Establish typography, color palettes, and visual hierarchy
- Design responsive layouts for mobile, tablet, and desktop
- Optimize conversion paths (hero → portfolio → inquiry)
- Design with visual storytelling in mind (videos should be front-and-center)

### 3. **Frontend Engineer Agent** — Implementation & Interactivity
**Role:**
Builds the actual website frontend based on designs. This agent will:
- Implement responsive HTML/CSS/JavaScript
- Integrate video players (Vimeo embeds)
- Build portfolio grid/filtering functionality
- Implement form submissions and contact flows
- Optimize performance (especially for video-heavy content)
- Ensure accessibility and semantic HTML

### 4. **Code Reviewer Agent** — Quality Assurance & Best Practices
**Role:**
Reviews all generated code for quality and standards. This agent will:
- Check for security vulnerabilities (especially form handling)
- Verify responsive design implementation
- Review code organization and maintainability
- Ensure performance best practices (lazy loading, image optimization)
- Validate accessibility standards (WCAG compliance)
- Check cross-browser compatibility

### 5. **Test Engineer Agent** — Functional & Integration Testing
**Role:**
Tests the built website to ensure everything works as intended. This agent will:
- Test form submissions and contact flows
- Verify video embeds and playback across devices
- Test responsive design on multiple screen sizes
- Validate navigation and page transitions
- Test social media integrations (Instagram feed)
- Performance testing (load times, especially with videos)
- Cross-browser testing

### 6. **Manager Orchestrator Agent** — Project Coordination (if needed)
**Role:**
Manages complex redesign projects requiring multiple agents working together. This agent will:
- Coordinate between UI/UX designers and frontend engineers
- Ensure design requirements are properly translated to code
- Manage handoffs between different agent specializations
- Track overall project progress and dependencies
- Escalate blockers or conflicting requirements

## Design Directions to Consider

**Visual Brand:**
- Modern, creative, cinematic
- Video/visual-first layout
- Dark theme option (common in video/creative industries)
- Bold typography and striking imagery
- Showcase their craftsmanship through the website itself

**Key Content Areas:**
1. **Hero Section** — embedded video showreel, compelling tagline
2. **Featured Work** — project gallery with Vimeo embeds, ability to play video
3. **Services** — clear breakdown of what they offer
4. **About** — team/behind-the-scenes content
5. **Testimonials** — client reviews and social proof
6. **Contact** — CTA for project inquiries

**Technical Considerations:**
- Vimeo API integration for project videos
- Instagram feed integration (real-time social proof)
- Optimized video loading and streaming
- SEO-friendly structure for portfolio projects
- Analytics tracking for inquiries/conversions
